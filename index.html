Control de versiones con GIT

Git es un sistema de control de versiones distribuido que permite a los desarrolladores colaborar en proyectos de software. Permite a los desarrolladores llevar un registro de los cambios en el código fuente, colaborar con otros desarrolladores y revertir a versiones anteriores del código.

Comandos Básicos de Git

Este contenido aborda los comandos básicos para empezar a trabajar con Git.

git help: Muestra la ayuda del comando que se le indique.

git init: Inicializa el repositorio local.

git status: Muestra la lista de archivos que se han cambiado y los que están por ser preparados o confirmados.

git add [archivo/patrón]: Añade el archivo a staging.

git commit -m 'mensaje': Guarda los archivos en el repositorio local con un mensaje descriptivo.

git log: Muestra el historial de commits.

Gestión de Ramas en Git

Este contenido cubre la creación, gestión y fusión de ramas en Git.

git branch: Se usa para listar, crear o borrar ramas.

git branch: Lista las ramas existentes.

git branch [nombre_nueva_rama]: Crea una nueva rama.

git branch -d [nombre_nueva_rama]: Elimina una rama existente.

git branch -m [old_nombre_rama] [new_nombre_rama]: Cambia el nombre de una rama.

git show: Permite ver el último commit al que está asociada una rama.

git log: Muestra los IDs de los commits del repositorio local.

git log -p: Incluye el historial de confirmaciones y cambios en los archivos.

git log --oneline: Muestra el historial de commits en una sola línea por commit.

git log --oneline --all: Muestra el historial de commits de todas las ramas.

git checkout: Te permite moverte entre ramas.

git checkout -b [nombre_nueva_branch]: Crea una nueva rama y cambia a ella automáticamente.

git checkout [ID_commit]: Permite volver a cualquier versión anterior en base al ID del commit.

git diff: Se usa para ver una lista de todos los conflictos presentes.

git diff [branch_origen] [branch_destino]: Ver los conflictos entre ramas antes de fusionarlas.

git merge: Se usa para fusionar una rama con otra rama activa.

git merge --abort: Descarta la fusión en caso de conflictos.

git log --merge: Muestra información sobre los commits que generaron un conflicto.

Clonación y Fetch en Git

Este contenido cubre los comandos de clonación, fetch y pull en Git.

git clone [remote_repo] [directory_local]: Copia un repositorio remoto en el entorno local.

git clone --branch [tag] [remote_repo]: Copia solo la versión de la etiqueta indicada.

git clone --branch --single-branch [branch_name] [remote_repo]: Copia solo la rama indicada.

git fetch: Actualiza el repositorio local con información del remoto, pero no fusiona cambios automáticamente.

git pull: Realiza un fetch y luego fusiona los cambios automáticamente.

Introduccion HTTP

HTTP (Hypertext Transfer Protocol) es un protocolo de comunicación entre un cliente y un servidor. Su principal propósito es la transferencia de información.

Funcionamiento del protocolo HTTP

Cliente WEB

  • Solicita información y la interpreta
  • Renderiza en pantalla el resultado
  • Interactúa con el usuario

Servidor WEB

  • Sirve contenido estático
  • Ejecuta programas para generar contenido dinámico
  • Certificados SSL/TLS

HTTP / HTTPS

HTTP

  • Es el protocolo de transferencia sin cifrado, esto es un problema de seguridad ya que en esta conexion los datos están expuestos.

HTTPS

  • Es el protocolo de transferencia con cifrado, esto es una solución al problema de seguridad de HTTP. Se necesita un certificado SSL/TLS para poder usar HTTPS.

Tipos de solicitudes

GET

  • Se usa para solicitar información al servidor sin modificar los datos en el servidor.

POST

  • Se usa para enviar información al servidor y modificar los datos en el servidor.

Codigos de estado

1XX

  • Respuestas informativas
    • 100: Continuar
    • 101: Cambio de protocolo

2XX

  • Respuestas satisfactorias
    • 200: OK
    • 201: Creado
    • 202: Aceptado

3XX

  • Redirecciones
    • 300: Múltiples opciones
    • 301: Movido permanentemente
    • 302: Encontrado

4XX

  • Errores del cliente
    • 400: Solicitud incorrecta
    • 401: No autorizado
    • 403: Prohibido
    • 404: No encontrado

5XX

  • Errores del servidor
    • 500: Error interno del servidor
    • 502: Error de puerta de enlace
    • 503: Servicio no disponible

Sistema de Nombres de Dominio (DNS)

El Sistema de Nombres de Dominio (DNS) es un sistema de nomenclatura jerárquico descentralizado para dispositivos conectados a redes IP como Internet o una red privada. Asocia información variada con nombres de dominio asignados a cada uno de los participantes.

Funcionamiento del DNS

Cliente

  • Realiza una petición de resolución de nombres de dominio.
  • Envía la petición al servidor DNS.

Servidor DNS

  • Recibe la petición de resolución de nombres de dominio.
  • Devuelve la dirección IP asociada al nombre de dominio solicitado.

Tipos de registros DNS

A

  • Asocia un nombre de dominio con una dirección IP.

AAAA

  • Asocia un nombre de dominio con una dirección IPv6.

CNAME

  • Asocia un nombre de dominio con otro nombre de dominio.

MX

  • Asocia un nombre de dominio con un servidor de correo.

NS

  • Asocia un nombre de dominio con un servidor de nombres.

TXT

  • Asocia un nombre de dominio con texto arbitrario.

Funciones del DNS

Resolución de nombres

  • Asocia un nombre de dominio con una dirección IP.

Redirección de correos

  • Asocia un nombre de dominio con un servidor de correo.

Redirección de tráfico

  • Asocia un nombre de dominio con un servidor web.

Servidor Web Apache

Apache es un servidor web de código abierto que se utiliza para servir sitios web. Es el servidor web más popular del mundo y es conocido por su flexibilidad y capacidad de personalización.

Funcionamiento de Apache

Cliente

  • Realiza una petición de un recurso web.
  • Envía la petición al servidor Apache.

Servidor Apache

  • Recibe la petición del cliente.
  • Devuelve el recurso solicitado al cliente.

Contextos del servidor

Contexto Servidor

  • En este contexto se definen las directivas que afectan a todo el servidor.

Contexto VirtualHost

  • En este contexto se definen las directivas que afectan a un host virtual (aplicacion web).

Contexto Directorio

  • En este contexto se definen las directivas que afectan a un directorio en concreto.
